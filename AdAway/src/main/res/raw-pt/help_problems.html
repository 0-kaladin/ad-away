Espaço insuficiente na partição

Try changing the target hosts file in preferences to /data/data/hosts (or /data/hosts) and apply AdAway again.

Does not work on Android 4.4+

Try changing the target hosts file in preferences from /data/data/hosts to /data/hosts or /system/etc/hosts and apply AdAway again.

Não bloqueia anúncios na aplicação XYZ!

Poderão faltar alguns hostnames no ficheiro de hosts fornecido pelas Origens de Hosts ou a aplicação poderá ter empacotado as imagens para apresentar anúncios sem aceder à internet.

Poderá registar pedidos de DNS (Menu -> Registar os pedidos de DNS) no AdAway para descobrir os hostnames adicionais que deverão ser bloqueados

Adicione os hostnames suspeitos à sua Blacklist com um toque longo nas entradas do registo e reporte estes hostnames quando os verificar no fórum Hosts Inbox em hosts-file.net.

Anúncios em cache

Por vezes as aplicações colocam anúncios em cache depois do download. Isto faz com que fiquem alguns anúncios nalgumas aplicações. Poderá tentar remover a cache destas aplicações na Lista de Aplicações do Android para resolver este problema.

A aplicação XYZ não se encontra a funcionar!

Algumas aplicações precisam de comunicar com um hostname que se encontra bloqueado pelo AdAway, ou recusar-se-ão a funcionar quando se encontram incontactáveis. Visite https://github.com/dschuermann/ad-away/wiki/ProblematicApps para ver uma lista de aplicações conhecidas com este problema. Caso contrário, verifique quais os hostnames necessários e adicione-os à sua Whitelist n'As suas listas e reporte-o no bug tracker do AdAway.

O botão de retroceder nos navegadores não funciona

Em alternativa, poderá activar o webserver local do AdAway nas preferências.
