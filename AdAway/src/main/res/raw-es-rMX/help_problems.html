No hay suficiente espacio en la partición

Intente cambiar el archivo de servidores web en las preferencias a /data/data/hosts e inicia AdAway de nuevo.

Does not work on Android 4.4+

Try changing the target hosts file in preferences from /data/data/hosts to /data/hosts or /system/etc/hosts and apply AdAway again.

¡No bloquea los anuncios en la aplicación XYZ!

Algunos nombres de los servidores web pueden faltar en los archivos proporcionados en Fuente de Servidores o la aplicación ha incorporado las imágenes para proporcionar anuncios sin acceso a Internet.

Puede registrar las peticiones DNS en (Menú-> Registro peticiones DNS) de AdAway para averiguar qué nombres de servidores web adicionales tienen que ser bloqueados.

Agregue los nombres de servidores web sospechosos a su propia lista negra manteniendo presionado las entradas en el Inicio de Sesión, y reporte estos nombres cuando se les han verificado en el Foro Buzón de Entrada de Servidores Web of hosts-file.net.

Anuncios en caché

A veces se almacenan aplicaciones de anuncios en caché después de su descarga. Esto conduce a anuncios en algunas aplicaciones. Usted puede tratar de borrar la memoria caché para estas aplicaciones en la lista de aplicaciones de Android para evitar este problema.

¡La aplicación XYZ ya no funciona!

Some applications need to communicate with a hostname that is blocked by AdAway or refuses to work when the hostnames which should serve ads are not reachable. See https://github.com/dschuermann/ad-away/wiki/ProblematicApps to get a list of known applications that have problems. Otherwise, find out which hostnames are needed and add them to your Whitelist under Your Lists and report them to the bug tracker of AdAway.

El botón de retroceso del navegador web no está funcionando

Puede activar servidor web local, de Adaway en las preferencias como una solución.
