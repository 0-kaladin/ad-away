Pas assez d'espace disque sur la partition

Essayer de remplacer le dossier cible du fichier hosts dans les préférences par /data/data/hosts et activer AdAway une nouvelle fois.

Les publicités dans l'application XYZ ne sont pas bloquées !

Certains noms de serveurs peuvent ne pas apparaitre dans la liste de serveurs fournie ou le programme a intégré les images afin de disposer de publicités sans accès internet.

Vous pouvez enregistrer les requetes DNS (Menu->Log DNS Requests) d'AdAway afin d'identifier les noms des serveurs a bloquer.

Ajouter les noms de serveurs suspects à votre propre liste noire en appuyant longuement sur les entrée du fichier log et informer le forum que vous les avez vérifiés Hosts Inbox of hosts-file.net.

Publicités en cache

Certaines applications téléchargent les publicités et les conservent en mémoire cache. C'est la raison pour laquelle, dans certaines applications, les publicités sont toujours présentes. Pour faire disparaître ces publicités, vous pouvez essayer de supprimer le cache de l'application en question grâce au gestionnaire d'applications Android : "Gérer les applications" > puis choisissez l'application > enfin dans les propriétés de l'application > cliquez sur le bouton "Vider le cache".

L'application XYZ ne fonctionne plus !

Certaines applications exigent une communication avec un serveur hôte qui est bloqué par AdAway ou refusent de fonctionner lorsque le serveur de publicités n'est pas disponible. Consultez https://github.com/dschuermann/ad-away/wiki/ProblematicApps pour avoir une liste d'applications connues ayant ce problème. Si vous parvenez à identifier les noms de ces serveurs , vous pouvez les ajouter à votre Liste Blanche sous Vos listes et les signalez à l'adresse bug tracker d'AdAway.

Dans votre navigateur web, le bouton Précédent ne fonctionne pas

Solution de contournement : activez le serveur web local dans les préférences d'AdAway .
