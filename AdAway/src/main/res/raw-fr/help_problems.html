Pas assez d'espace sur la partition

Essayer de remplacer le fichier cible « hosts » dans les préférences par /data/data/hosts et relancer AdAway.

Does not work on Android 4.4+

Try changing the target hosts file in preferences from /data/data/hosts to /data/hosts or /system/etc/hosts and apply AdAway again.

Les publicités dans l'application XYZ ne sont pas bloquées !

Certains noms d'hôte peuvent manquer dans les fichiers « hosts » fournis par les sources « hosts » ou l'application a intégré les images afin de proposer des pubs sans accéder à internet.

Vous pouvez journaliser les requêtes DNS (Menu->Tracer les requêtes DNS) d'AdAway afin d'identifier quels autres noms d'hôte doivent être bloqué.

Ajouter les noms d'hôte suspects à votre propre liste noire en appuyant longuement sur les entrées du journal et rapporter les sur le forum après vérification Hosts Inbox de hosts-file.net.

Publicités en cache

Parfois les applications mettent en cache les publicités après le téléchargement, et des publicités restent dans certaines applications. Vous pouvez essayer de supprimer le cache de ces applications dans le gestionnaire d'applications d'Android pour contourner ce problème.

L'application XYZ ne fonctionne plus !

Certaines applications on besoin de communiquer avec un nom d'hôte qui est bloqué par AdAway ou refusent de fonctionner quand les noms d'hôte qui servent les pubs ne sont pas joignables. Voir https://github.com/dschuermann/ad-away/wiki/ProblematicApps pour obtenir une liste d'applications connues qui posent des problèmes. Autrement, identifier quels noms d'hôte sont nécessaires, ajouter les à votre Liste blanche sous Vos listes et rapportez les au gestionnaire de bogues d'AdAway.

Le bouton retour des navigateurs Web ne fonctionne pas

Vous pouvez activer le serveur Web local d'AdAway dans les préférences comme palliatif.
