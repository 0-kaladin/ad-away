Próbáld meg módosítani a cél hosts file lehetőségeit /data/data/hosts, és újra alkalmazni az AdAway-t.

Nem blokkolja a hirdetéseket XYZ alkalmazásban!

Néhány hostnévnév hiányozhat a megadott host fájlból a Hosts Forrásokban, vagy az alkalmazás tartalmazza a hirdetések képeit, így reklámoz internet elérése nélkül.

Lehet naplózni az AdAway DNS kéréseit (Menü->DNS kérések naplózása), hogy további blokkolandó host neveket keressünk benne.

Adja a gyanús host neveket a saját Fekete listájához - hosszan nyomva tartva a naplóban -, majd jelentse ezeket a host neveket a hosts-file.net fórum Hosts Inbox részében, ha meggyőződött róla, hogy tényleg reklámot szolgáltatnak.

Gyorstárazott reklámok

Néhány alkalmazás gyorstárazza a reklámokat letöltés után. Így reklámok maradhatnak egyes alkalmazásokban. Megpróbálhatja kitörölni ezen alkalmazások gyorstárát az Android Alkalmazások listájában, hogy ezek a reklámok is eltűnjenek.

Az XYZ alkalmazás már nem működik!

Some applications need to communicate with a hostname that is blocked by AdAway or refuses to work when the hostnames which should serve ads are not reachable. See https://github.com/dschuermann/ad-away/wiki/ProblematicApps to get a list of known applications that have problems. Otherwise, find out which hostnames are needed and add them to your Whitelist under Your Lists and report them to the bug tracker of AdAway.

The back button in web browsers is not working

You can enable AdAway's local webserver in preferences as a workaround.
