分割區沒有足夠的空間

請嘗試在設定中更改目標 hosts 檔為 /data/data/hosts 及再次套用到 AdAway。

AdAway 未能在某些應用程式中攔截廣告！

有些主機名稱可能在 Hosts 來源所提供的 hosts 檔中沒有收錄，又或者應用程式本身已捆綁內建的廣告圖像，所以應用程式就算無連接到網絡也可顯示廣告。

您可使用 AdAway 的記錄 DNS 請求功能 (選單 -> 記錄 DNS 請求) 去尋找更多可以攔截的主機名稱。

只要在記錄中，長按那些可疑的主機名稱就可加至您的黑名單內。同時您可到 hosts-file.net 中的論壇子分區 Hosts Inbox 頁面去驗證那些可疑的主機名稱，如果驗證屬實同時可在該區進行回報。

儲存於快取中的廣告

Sometimes applications cache advertisements after download. This leads to remaining advertisements in some applications. You can try to delete the cache for these applications in Android's Application list to circumvent this problem.

當使用 AdAway 及開啟廣告攔截後，某些應用程式就不能正常運作甚至無法運作！

Some applications need to communicate with a hostname that is blocked by AdAway or refuses to work when the hostnames which should serve ads are not reachable. See http://code.google.com/p/ad-away/wiki/ProblemApps to get a list of known applications that have problems. Otherwise, find out which hostnames are needed and add them to your Whitelist under Your Lists and report them to the bug tracker of AdAway.
